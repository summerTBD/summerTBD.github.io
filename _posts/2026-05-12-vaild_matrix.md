### 题目来源

# 合法矩阵最小修改次数

## 题目描述

小红定义一个 01 矩阵是合法的，当且仅当其中每个由 0 组成的四连通块大小都恰好为 3，并且每个由 1 组成的四连通块大小也都恰好为 3。

现在给定一个大小为 $2 \times n$ 的 01 矩阵。你每次可以修改一个字符，将 1 改成 0，或将 0 改成 1。

请你求出，最少需要修改多少个字符，才能使这个矩阵变成合法矩阵。如果无论如何都无法变成合法矩阵，输出 $-1$。

这里的连通块采用四连通定义，也就是说两个格子只有在共边时才算相邻。

## 输入描述

第一行输入一个整数 $n$（$1 \le n \le 2 \times 10^5$）。

接下来两行，每行输入一个长度为 $n$ 的 01 字符串，表示矩阵的两行。

## 输出描述

输出一个整数，表示最少修改次数。

如果无解，输出 $-1$。

## 示例 1

**输入**
```
6
110110
100100
```

**输出**
```
0
```

**说明**
原矩阵本身已经合法。

其中左侧的 3 个 1 构成一个大小为 3 的连通块，右侧的 3 个 1 也构成一个大小为 3 的连通块。所有 0 的连通块大小同样都恰好为 3，因此不需要修改。

### 初步探路

在看到这道题，我最初想到的对它进行转换，把1->1,0->-1，为什么这么做？这来源于一道codeforce上的一道题，Codeforces Round 1097 (Div. 2, Based on Zhili Cup 2026)的C题，不过这其实和这道题没什么关系，首先很容易知道3|n，我们接下来就要考虑它的形状了，在最初我并没有注意到一点，那就是横着的111或者000也是符合要求的连通块，这很重要。在最初我只考虑了L型的块。如果是这样，我们需要考虑每个单独的块该怎样摆，注意，我们不能直接从其中某一块单独考虑，像这样其实根本不存在独立的块，我们要从第一个考虑，如果真是我们想的那样，每一列的和其实是-1,0,1，-1，0，1...或者1,0，-1，1,0，-1...不断重复的情况，不过这很糟糕，你会发现，这样的思考我们忽略一些可能的情况，因为我们遗漏了每一列都是0的情况，比如说上层是111000而下层是000111，根据我们的映射规则和推论你会得到一个错误的答案，核心原因就在于我们忽略了，横着摆也是合法的。

### 补全思考

现在我们已经知道了，一个块要么是“L”形要么是“---”型。这意味着我们映射法则是不行的。不过我们刚才的思路并不是毫无用处，由于其实不存在真正独立的块，所以我们还是可以考虑从第一个入手，不过其实很容易想到一个问题，我们只考虑一个块其实对我们没有任何帮助，应该是两个块，为什么？还记得我们遗漏的"---"型吗，我们很容易发现，在要考虑这种“---”型下，天然就要考虑两个块，如果我们把两个块合在一起，在不考虑左右时候，有哪些情况呢？分别有6种，a1: top"110" bot"100"，a2: top"100" bot"110",b1: top"011" bot"001",b2: top"001" bot"011",c1: top"111"bot"000",c2:top"000"bot"111".这里你可以看到6种情况，我却只用了3个字母，可以先思考，为什么这么分？
还是根据我们的原则，从第一个开始看，如果是a1，那第二个只能是a1或者a2，如果是a2第二个只能是a1，或者a2，（这里其实可以根据a1和a2是倒过来的，便可以根据对称性快速得到），同理b1的后面只能是b1或者b2,b2后面只能是b1或者b2.不过对于c1后面只能是c2，c2后面只能是c1，那这我们就可以发现，整个式子只能存在a1和a2的混合或者b1和b2的混合，或者c1和c2的交替（不是混合）。那我们就可以直接枚举每一个2*3变成合法matrix的步骤，最后在相加，比如，在只有a1和a2的式子，由于每一个2*3都是a1和a2二选一，我们可以暴力判断谁更少，我们直接选择最小的就行，其它的同理

### 代码

#include<bits/stdc++.h>
using namespace std;

int main(){
    ios::sync_with_stdio(false);
    cin.tie(0);
    
    int n;
    cin >> n;
    string s1, s2;
    cin >> s1 >> s2;
    
    if(n % 3){
        cout << -1 << '\n';
        return 0;
    }
    
    auto calc = [&](int left, const string& top, const string& bot){
        int res = 0;
        for(int k = 0; k < 3; ++k){
            if(s1[left + k] != top[k]) res++;
            if(s2[left + k] != bot[k]) res++;
        }
        return res;
    };
    
    int m = n / 3;
    int costA = 0, costB = 0, costC0 = 0, costC1 = 0;
    
    for(int i = 0; i < m; ++i){
        int l = i * 3;
        
        int a1 = calc(l, "110", "100");
        int a2 = calc(l, "100", "110");
        
        costA += min(a1, a2);
        
        int b1 = calc(l, "011", "001");
        int b2 = calc(l, "001", "011");
        
        costB += min(b1, b2);
        
        int c1 = calc(l, "111", "000");
        int c2 = calc(l, "000", "111");
        
        if (i%2 == 0) {
            costC0 += c1;
            costC1 += c2;
        } else {
            costC0 += c2;
            costC1 += c1;
        }
        
    }
    
    int ans = min({costA, costB, costC0, costC1});
    cout << ans << '\n';
    
    return 0;
}
