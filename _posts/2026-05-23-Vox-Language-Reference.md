## 1. 设计原则

1.  **显式优先，可控可免**：所有行为优先在代码字面上可见。完全可控、可预测的隐式（如关键字引导的默认行为）不在禁止之列。不存在隐式类型转换、隐式构造函数、隐式 `self`。
2.  **正交优于合并**：每个特性只做一件事，且彼此独立。你可以自由组合它们，而不是被迫接受一个"全家桶"。
3.  **无魔法**：语言不提供任何需要"背后特殊处理"的语法。所有高级语法（如 `try`、方法调用）都可通过明确的脱糖规则还原为基础语法。
4.  **控制权归程序员**：内存何时分配、何时释放、数据如何布局，都由你决定。语言提供工具（如 `defer`），但绝不替你作主。
5.  **无必要不增实体**：不会因为"别的语言有"就加一个特性。每个特性必须解决真实痛点，且不能被已有特性轻易替代。
6.  **性能可预测**：不存在"看似轻量，实则触发隐式分配"的操作。每一行代码的时间与空间成本，都应能从字面上合理推断。

## 2. 词法结构

### 2.1 关键字
以下为保留关键字，不可用作标识符。

```
const var fn struct enum match
if else while for return yield break continue
defer import pub extern
as parse null true false
and or not
test macro comptime asm global
```

### 2.2 标识符
由字母、数字、下划线组成，必须以字母或下划线开头。不允许数字开头。

### 2.3 字面量
- **整数字面量**：十进制 `42`，十六进制 `0x2A`，八进制 `0o52`，二进制 `0b101010`，可加下划线 `1_000_000`。
- **浮点字面量**：`3.14`，`1.0e-9`。
- **字符串字面量**：用双引号括起，支持转义 `\n`、`\t`、`\\`、`\"`，类型为 `[]const u8`。
- **布尔字面量**：`true`、`false`。
- **空指针字面量**：`null`，类型可赋值给任意 `*T` / `*mut T` 以及 `&T` / `&mut T`。
- **单元字面量**：`()`，类型 `unit`。

### 2.4 运算符与分隔符
运算符及其优先级见[附录](#19-附录)。  

**逻辑运算符**（Python 风格）：`and` `or` `not`（关键字）  
**位运算符**：`&` `|` `^` `~`  
**引用运算符**：`&` `&mut`  
**分隔符**：`{ } ( ) [ ] ; , . => : ..`

### 2.5 注释
- 单行注释：`//` 至行尾。
- 多行注释：`/* ... */`，可嵌套。

## 3. 基本语法

### 3.1 语句与表达式
- **语句语境**：独立语句、循环体、`if` 作为语句时等，不产生值。  
- **求值语境**：赋值右值、函数实参、`return` 右值、`match` 臂等，必须用 `yield`（块内）或 `return`（函数）提供值。

### 3.2 分号
**语句必须以分号结尾**。表达式语句、变量声明、赋值、`return`、`yield`、`defer`、`global` 均需分号。块 `{}` 之后通常不加分号，除非块作为表达式语句的一部分。

## 4. 变量与声明

### 4.1 不可变变量与可变变量

```rust
// 不可变变量（默认）
const x: i32 = 10;
const y = 20;       // 类型推导

// 可变变量
var counter: i32 = 0;
counter = 1;        // 允许重新赋值
x = 5;              // 错误：不可变变量不可赋值
```

- `const` 声明必须初始化，且后续不可修改绑定。
- `var` 声明允许重新赋值，但类型必须匹配。

### 4.2 变量遮蔽

Vox 支持**跨作用域遮蔽**（内层作用域可以声明与外层同名的变量），但**禁止同作用域重复声明**。

```rust
var x: i32 = 5;

// 跨作用域遮蔽：允许
if cond {
    var x: f64 = 3.14;   // 合法：新作用域，新变量
}

// 同作用域重复声明：禁止
var x: i32 = 10;         // 错误：标识符 'x' 已在当前作用域声明
```

**设计理由**：同作用域遮蔽会使旧变量隐式"消失"，与显式原则相悖。若需要重新绑定，请使用新名字或块表达式隔离中间变量。

### 4.3 全局变量

- 模块顶层的 `const` 若无循环依赖，应在编译期求值，值存入只读数据段。
- 模块顶层的 `var` 在进入 `main` 前初始化，初始化顺序按依赖拓扑排序。跨模块的可变全局状态由程序员自行保证同步安全。

#### 4.3.1 全局变量的函数内访问 (`global`)

默认情况下，函数内部无法直接读写模块顶层的变量——它们必须被显式引入。  
使用 `global` 语句可以解除这一限制：

```rust
// 模块顶层
var counter: i32 = 0;

fn increment() {
    global counter;       // 声明我要用外部的 counter
    counter = counter + 1;
}

fn read_counter() -> i32 {
    global counter;
    return counter;
}
```

**批量声明**：可以在一个 `global` 语句中声明多个变量，用逗号分隔。
```rust
global counter, MAX, app_name;
```

**规则**：
- `global` 只能用于当前模块顶层已声明的 `const` 或 `var`。
- 不能通过 `global` 修改 `const` 变量。
- 从其他模块导入的变量（如 `math.PI`）使用模块路径访问，不需要 `global`。
- 若函数内未使用 `global` 声明，直接对与全局变量同名的标识符赋值，将视为对未定义变量的错误。
- `global` 声明本身不分配内存，不产生任何运行时开销——它是纯粹的编译期名字绑定指令，翻译后消失。

## 5. 类型系统

### 5.1 基础类型
| 类型 | 描述 |
| :--- | :--- |
| `i8`、`i16`、`i32`、`i64` | 有符号整数 |
| `u8`、`u16`、`u32`、`u64` | 无符号整数 |
| `f32`、`f64` | 浮点类型 |
| `bool` | 布尔型 (`true`, `false`) |
| `unit` | 单元类型，字面量 `()`，仅一个值 |

### 5.2 复合类型
- **定长数组**：`[T; N]`，详见第 12 节。
- **切片**：`[]T` 和 `[]const T`，详见第 12 节。
- **引用**：`&T`（不可变引用）、`&mut T`（可变引用）。详见第 10 节。
- **指针**：`*T`（指向不可变数据）、`*mut T`（指向可变数据）。另见第 11 节。
- **结构体**：`struct { ... }`，详见第 8 节。
- **枚举**：`enum Name { Variant1, Variant2(负载类型), ... }`，详见第 9 节。

### 5.3 可选类型
标准库定义 `Option<T>` 枚举：
```rust
enum Option<T> {
    Some(T),
    None,
}
```
任何可能缺失的值应使用 `Option<T>` 表达。

### 5.4 类型转换
- **`as` 操作符**：用于安全的位模式转换或数值宽化/窄化。`"123" as i32` 是编译错误。
- **`parse` 函数**：将字符串解析为数值，返回 `Result<T, ParseError>`。

### 5.5 类型推导
允许在初始化时省略类型标注，推导范围限定为局部变量和函数返回值。函数参数必须显式标注。

## 6. 函数

```rust
fn add(a: i32, b: i32) -> i32 {
    return a + b;
}
```

- 函数体是求值语境，**必须显式使用 `return` 提供返回值**。
- 所有执行路径都必须遇到 `return`，否则编译错误。
- 不支持函数重载。Vox 鼓励用**明确的函数名**或**结构体封装参数**代替重载。
- 函数参数默认不可变，如需可变需在参数名前加 `var`（例如 `fn change(var x: i32)`）。

### 6.1 返回类型推导
可从第一个 `return` 语句推导返回类型，但推荐显式标注。

### 6.2 参数过多问题的推荐解法
当函数参数超过 3–4 个时，推荐将相关参数组织进一个结构体：

```rust
struct Config {
    host: []const u8,
    port: u16,
    timeout: u32,
}

fn connect(cfg: Config) -> Result<Connection, Error> {
    // ...
}
```
这种方式显式、自文档化，且不引入任何额外运行时开销。

## 7. 控制流

### 7.1 `if` 表达式 / 语句
`if` 可作为语句或表达式，取决于所处语境。
- 作为**语句**：不要求所有分支产生值，不要求必有 `else`。
- 作为**表达式**（求值语境）：`if` 必须包含 `else`，且每个分支必须用 `yield` 提供值。

```rust
// if 作为表达式
var status = if cond {
    yield 1;
} else {
    yield 0;
};

// if 作为语句
if flag {
    do_work();
}
```

### 7.2 `while` 循环（语句）
`while` 是语句，不返回值。
```rust
while i < 10 {
    i = i + 1;
    if i == 5 {
        continue;
    }
    if i == 8 {
        break;
    }
}
```

### 7.3 `for` 循环（语句）
`for` 也是语句，不返回值。
```rust
for item in collection {
    print(item);
}

for var idx in 0..len {
    arr[idx] = idx;
}
```
迭代变量默认 `const`，如需可变用 `for var`。`break` 和 `continue` 可用标签。

### 7.4 标签化的 `break` / `continue`
```rust
'outer: for i in 0..10 {
    for j in 0..10 {
        if matrix[i][j] == target {
            break 'outer;
        }
    }
}
```

### 7.5 `yield` 表达式
用在**求值语境下的内嵌块**（`if`/`match`/独立块）中，提供块的值并退出该块。
- 如果在语句语境中出现 `yield`，编译错误。

### 7.6 独立块表达式
```rust
var x = {
    var temp = compute();
    yield temp + 1;
};
```
该块必须在所有出口用 `yield` 提供值。

## 8. 结构体与方法

```rust
struct Point {
    x: f64,
    y: f64,
}

// 方法定义：第一个参数显式为 self
fn Point.length(self: &Point) -> f64 {
    return sqrt((self.x * self.x + self.y * self.y) as f64);
}
```

- 方法归属 struct 命名空间，通过 `Type.method()` 调用。
- 无函数重载，无隐式 `self`。
- `self` 参数可以是 `&T`（不可变方法）、`&mut T`（可变方法）或 `*T` / `*mut T`（底层指针方法）。
- 调用语法糖：`p.length()` 脱糖为 `Point.length(&p)`。
- 若方法需要修改结构体，使用 `self: &mut T`：
  ```rust
  fn Point.set_x(self: &mut Point, val: f64) {
      self.x = val;
  }
  ```

## 9. 枚举与模式匹配

### 9.1 枚举声明
```rust
enum OptionInt {
    Some(i32),
    None,
}
```
变体可携带负载类型。

### 9.2 `match` 表达式
- `match` 是表达式，必须位于求值语境或作为语句都可以（语句时各分支可无 `yield`）。
- 在求值语境下，每个分支的块必须以 `yield` 提供值。
- **穷尽性**：编译器强制匹配所有变体，否则编译错误。

```rust
var result = match opt {
    Some(v) => {
        yield v * 2;
    }
    None => {
        yield 0;
    }
};
```

## 10. 引用与借用

### 10.1 引用类型

Vox 提供两种引用类型，与裸指针并列但**自带编译器安全检查（借用检查）**：

| 类型 | 含义 | 可读 | 可写 |
| :--- | :--- | :---: | :---: |
| `&T` | 不可变引用 | ✅ | ❌ |
| `&mut T` | 可变引用 | ✅ | ✅ |
| `*T` | 不可变裸指针 | ✅（不检查） | ❌ |
| `*mut T` | 可变裸指针 | ✅（不检查） | ✅（不检查） |

引用的核心契约："借用"——你暂时借走访问权，但不拥有数据，用完后必须归还。

### 10.2 创建与解引用

```rust
const data: i32 = 42;

// 创建引用
const r: &i32 = &data;          // 不可变引用
var val: i32 = 10;
var rm: &mut i32 = &mut val;    // 可变引用

// 解引用
var x: i32 = *r;                // 通过引用读取
*rm = 99;                       // 通过可变引用写入
```

- `&expr` 创建不可变引用，`&mut expr` 创建可变引用。
- 解引用操作符 `*` 用于通过引用访问其所指向的值。
- 切片引用同理：`&arr[0..3]` 的类型是 `&[]i32`。

### 10.3 借用规则

编译器强制以下规则，确保引用不会导致数据竞争或悬空访问：

**规则 1（读写分离）**：任意时刻，对于一个值，要么存在**任意多个** `&T` 引用，要么存在**唯一一个** `&mut T` 引用，不能同时存在。

```rust
var data: i32 = 42;

const a: &i32 = &data;          // ✅ 不可变引用 1
const b: &i32 = &data;          // ✅ 不可变引用 2（与 a 共存）
print(*a, *b);                   // ✅ 多个只读读者，安全

var c: &mut i32 = &mut data;    // ❌ 编译错误：已有不可变引用活跃
*c = 99;
```

```rust
var data: i32 = 42;

var rw: &mut i32 = &mut data;   // ✅ 唯一可变引用
*rw = 99;

const r: &i32 = &data;          // ✅ 此时 rw 已不再使用，可创建新的不可变引用
```

**规则 2（原值冻结）**：当 `&mut T` 引用存在且活跃时，**原值被冻结**——不可通过原值路径读或写。

```rust
var data: i32 = 42;

var rw: &mut i32 = &mut data;   // rw 借走了 data 的可变访问权
print(data);                     // ❌ 编译错误：data 已被可变借用冻结
*rw = 99;                        // ✅ 只能通过借用者访问
```

**规则 3（存活期不超过被借者）**：引用的生命周期不能超过被引用变量的生命周期。

```rust
var r: &i32;                     // 声明一个引用变量

{
    var local: i32 = 10;
    r = &local;                  // ❌ 编译错误：local 离开作用域后 r 将悬空
}

print(*r);
```

### 10.4 引用与裸指针的选择

| 场景 | 推荐使用 | 理由 |
| :--- | :--- | :--- |
| 想要编译器保障安全 | `&T` / `&mut T` | 借用检查器防止悬空和数据竞争 |
| 需要指针运算 | `*T` / `*mut T` | 引用不提供算术运算 |
| 与 C 代码交互 | `*T` / `*mut T` | 直接映射 C 指针 |
| 实现自定义分配器 | `*mut T` | 底层内存操作需要完全控制 |
| 对 null 语义的需求 | `*T` / `*mut T` | 引用不可为 null |

引用和裸指针可以自由互转——这种转换是显式的，编译器不做隐藏的转换：

```rust
const data: i32 = 42;
const r: &i32 = &data;
const p: *i32 = r as *i32;       // 引用 → 裸指针：显式转换
const r2: &i32 = &*p;            // 裸指针 → 引用：需确保此时 p 仍有效
```

### 10.5 函数中的引用

引用可以作为函数参数和返回值类型：

```rust
// 读取值（不可变借用）
fn get_value(data: &i32) -> i32 {
    return *data;
}

// 修改值（可变借用）
fn increment(data: &mut i32) {
    *data = *data + 1;
}

// 返回引用
fn first(arr: &[i32]) -> &i32 {
    return &arr[0];
}
```

**多个引用参数时的生命周期省略规则**（由编译器自动推断，无需手动标注）：
- 如果只有一个引用参数，其生命周期自动关联到返回值的引用。
- 如果有多个引用参数，且返回值是引用类型，编译器要求显式标注以消除歧义（未来版本可能引入简化标注语法）。

### 10.6 方法中的 self 引用

方法定义时，`self` 参数的类型决定了该方法是否允许修改结构体：

```rust
struct Counter {
    value: i32,
}

// 不可变方法：只能读取
fn Counter.read(self: &Counter) -> i32 {
    return self.value;
}

// 可变方法：可以修改
fn Counter.add(self: &mut Counter, delta: i32) {
    self.value = self.value + delta;
}
```

### 10.7 引用与 defer 的关系

引用和 `defer` 是**两个独立的机制，各司其职**：

- **引用（`&T` / `&mut T`）**：管理"访问权"——谁能读、谁能写、能借用多久。
- **`defer`**：管理"资源"——文件、堆内存、锁等资源何时释放。

两者互补，互不替代：

```rust
fn process_with_resource() {
    var buf: *mut u8 = alloc.allocate(1024);
    defer alloc.deallocate(buf);      // defer 管释放

    var view: &[]u8 = &buf[0..512];   // 引用管访问安全
    process(view);
    // view 超出作用域，借用结束
    // 但 buf 直到函数退出才由 defer 释放
}
```

### 10.8 引用的转译策略（到 C）

引用在转译到 C 时，**直接映射为同类型的指针**，零运行时开销：

| Vox | C |
| :--- | :--- |
| `&T` | `T*`（const 限定，若原类型不可变） |
| `&mut T` | `T*` |
| `&expr` | `&expr`（C 的取地址） |
| `*r`（解引用） | `*r`（C 的解引用） |

借用检查是纯编译期机制，转译后完全消失，不影响生成的 C 代码。

## 11. 指针与内存管理

### 11.1 裸指针
- `*T`：指向不可变数据的指针。
- `*mut T`：指向可变数据的指针。
- 指针运算：`+`、`-`（步长为指向类型大小），`*` 解引用。
- 指针可为 `null`，用 `null` 字面量产生空指针。

### 11.2 堆分配
标准库提供分配器对象 `std_heap`，底层为 `malloc`/`free` 映射。
```rust
var alloc = std_heap;
var mem: *mut u8 = alloc.allocate(1024);
defer alloc.deallocate(mem); // 作用域出口自动释放
```

### 11.3 `defer` 语句
`defer` 后跟函数调用或块，在所在作用域退出前按后进先出顺序执行。纯编译期语法糖，零运行开销。

```rust
fn process() {
    var f = open("data.txt");
    defer close(f);

    var mem = alloc.allocate(256);
    defer alloc.deallocate(mem);
    // 无论函数如何退出，两个 defer 都会逆序执行
}
```

### 11.4 内存安全哲学
Vox 提供**渐进式安全**，程序员可以自由选择安全级别：

| 级别 | 使用方式 | 保护程度 | 适用场景 |
| :--- | :--- | :---: | :--- |
| 安全 | `&T` / `&mut T` + `defer` | 高（编译器检查） | 业务逻辑 |
| 中等 | `&T` / `&mut T` + 手动管理 | 中（只防悬空引用） | 性能敏感区 |
| 自由 | `*T` / `*mut T` + `defer` | 低（程序员保证） | 底层系统编程 |
| 裸 | `*T` / `*mut T` | 无（完全手动） | 自定义分配器、FFI |

语言本身**不强制**任何安全级别，程序员通过选择类型来选定保护程度。未来可通过编译选项全局收紧安全检查。

## 12. 数组与切片

### 12.1 定长数组 `[T; N]`

```rust
var arr: [i32; 5];                     // 声明，未初始化
var arr: [i32; 5] = [1, 2, 3, 4, 5];   // 声明并初始化
const arr = [1, 2, 3];                 // 类型推导为 [i32; 3]
```

- 定长数组是值类型，赋值会复制整个数组。
- 索引越界是未定义行为，由程序员自行保证。
- 数组字面量必须提供与声明大小匹配的元素数量。

**访问**：
```rust
arr[0] = 10;
var x = arr[1];
```

### 12.2 切片 `[]T` 和 `[]const T`

```rust
var slice: []i32 = arr[0..3];          // 可变切片，可修改元素
var view: []const i32 = arr[0..3];     // 不可变切片，只读
```

- 切片是胖指针，包含数据指针和长度。
- 字符串字面量 `"hello"` 的类型为 `[]const u8`。
- `[]T` 的元素可修改，`[]const T` 的元素只读。
- 切片操作 `arr[start..end]` 返回一个切片，左闭右开。

### 12.3 切片引用

可通过引用来借用切片的访问权，遵循借用规则：

```rust
var data: [i32; 5] = [1, 2, 3, 4, 5];

const full: &[i32] = &data;            // 对整个数组的不可变引用
const part: &[i32] = &data[1..4];      // 对切片子集的不可变引用
var mut_part: &mut [i32] = &mut data[0..2];  // 可变引用
```

## 13. 输入输出

Vox 提供两个内置符号用于基本的控制台交互。它们不是关键字，而是编译器自动导入的函数。

### 13.1 输出

```rust
print("Hello, ");
print("World!\n");
print("The answer is ", 42, "\n");
```

- `print` 接受任意数量的参数，按顺序输出到标准输出。
- 参数类型必须为 `[]const u8`、整型、浮点型或布尔型。
- **不自动追加换行符**。换行由程序员通过 `\n` 显式控制。
- 不提供格式化占位符。需要格式化时，使用字符串拼接或未来标准库中的格式化模块。

### 13.2 输入

```rust
var line = input();
```

- `input()` 从标准输入读取一行，返回 `[]const u8`。
- 返回的字符串不包含末尾的换行符。
- 若需要将输入解析为数值，使用 `parse` 函数：

```rust
var n = parse(input()) as i32;
```

### 13.3 实现说明

`print` 和 `input` 在转译到 C 时，直接映射到 C 标准库的 `printf` / `puts` 和 `fgets`，无额外运行时开销。

## 14. 模块系统

**文件即模块**：每个 `.vox` 文件就是一个模块。
- **`pub` 关键字**：声明公有符号（函数、类型、变量）。
- **`import` 语句**：导入其他模块。

**示例：**
```rust
// math.vox
pub const PI = 3.14159;
pub fn surface(r: f64) -> f64 {
    return PI * r * r;
}

// main.vox
import math;
print(math.surface(10.0));
```
- 导入模板：`import module_name`，之后使用 `module_name.member` 访问。
- 模块路径对应文件系统：`import math.geometry` 查找 `math/geometry.vox`。
- 模块通过文件路径隐式形成命名空间，无需手动声明。

## 15. 错误处理

标准库定义 `Result<T, E>` 枚举：
```rust
enum Result<T, E> {
    Ok(T),
    Err(E),
}
```

- 推荐函数在可能失败时返回 `Result`。
- 提供 `try` 操作符（语法糖），若表达式结果为 `Err` 则立即从当前函数返回该错误：

```rust
fn read_config() -> Result<Config, Error> {
    var data = try read_file("config.json");
    return parse_config(data);
}
```

## 18. 语法糖与脱糖规则

### 18.1 方法调用
`obj.method(args)` 脱糖为 `Type.method(&obj, args)`（若 `self: &T` 或 `self: *T`）或 `Type.method(&mut obj, args)`（若 `self: &mut T` 或 `self: *mut T`）。

### 18.2 索引操作
`a[b]` 脱糖为 `a.at(b)`（若用于读取）或 `a.at_mut(b)`（若用于赋值）。因此自定义类型只需实现 `at` / `at_mut` 方法即可支持下标语法。

### 18.3 `try` 表达式
`try expr` 脱糖为：
```
match expr {
    Ok(val) => val,
    Err(e) => return Err(e),
}
```

### 18.4 Lambda

Lambda 是纯编译期语法糖，转译到 C 后有明确的映射规则。现代 C 编译器优化能力极强，闭包结构体和函数指针调用通常可被完全内联优化，最终二进制等价于手写 C。

#### 18.4.1 Lambda 语法

Lambda 使用 `fn` 关键字定义，可选捕获列表声明对外部变量的捕获方式。

捕获列表 `[...]` 位于参数列表之前，可包含**默认捕获关键字**和**变量捕获项**：

| 写法 | 含义 |
| :--- | :--- |
| `[copy]` | 默认**值捕获**——lambda 体内所有用到的外部变量以副本形式捕获 |
| `[ref]` | 默认**引用捕获**——lambda 体内所有用到的外部变量以引用形式捕获 |
| `[base]` | 指定 `base` 以值捕获 |
| `[&config]` | 指定 `config` 以不可变引用捕获 |
| `[&mut counter]` | 指定 `counter` 以可变引用捕获 |
| `[]` 或无捕获列表 | 不捕获任何外部变量（纯函数） |

默认捕获关键字和变量捕获项可混合使用，变量项覆盖默认值：

```rust
// 无捕获列表 — 纯函数
var f = fn (x: i32) -> i32 { return x * 2; };

// 指定捕获
var g = fn [base] (x: i32) -> i32 { return x + base; };               // 值捕获
var h = fn [&config] () -> i32 { return config.value; };               // 引用捕获
var i = fn [&mut counter] () -> i32 { counter = counter + 1; };        // 可变引用捕获

// 默认捕获关键字（编译器推断捕获哪些变量）
var j = fn [copy] (x: i32) -> i32 { return x + base + counter; };      // 默认全部值捕获
var k = fn [ref] () -> i32 { return config.value + counter; };         // 默认全部引用捕获

// 混合：默认 + 个别覆盖
var l = fn [ref, timeout] () -> i32 {                                   // 默认引用，但 timeout 值捕获
    return config.value + timeout;
};
var m = fn [copy, &mut counter] () -> i32 {                            // 默认值拷贝，但 counter 可变引用捕获
    counter = counter + config.step;
    return counter;
};
```

**规则**：
- 每个捕获项为关键字 `copy`/`ref`（默认模式）、`name`（值捕获）、`&name`（不可变引用捕获）或 `&mut name`（可变引用捕获）。
- 捕获列表中最多出现一个默认捕获关键字（`copy` 或 `ref`），且必须位于列表首位。
- 无捕获列表时（即无 `[...]` 或 `[]`），lambda 体内只能使用参数和局部变量，不能引用外部变量。
- 值捕获存储变量在闭包创建时的快照（副本），引用捕获存储指向外部变量的引用。
- 默认捕获关键字（`[copy]`/`[ref]`）是一种**可控的隐式**：捕获的变量列表由编译器根据 lambda 体自动推断，但捕获方式由程序员显式指定。读者一眼可知"这个 lambda 默认值捕获/引用捕获"。
- 引用捕获的 lambda 必须在其捕获变量的生命周期内使用，编译器在借用检查阶段验证这一点。

#### 18.4.2 无捕获 lambda → 函数指针

当 lambda 没有捕获列表（不捕获任何外部变量）时，转译为独立的 C 函数，类型映射为函数指针。

```rust
var f = fn (x: i32) -> i32 { return x * 2; };
var result = f(21);
```

```c
int32_t lambda__1(int32_t x) { return x * 2; }
int32_t (*f)(int32_t) = &lambda__1;
int32_t result = f(21);
```

零运行时开销——调用方式与直接函数调用完全一致。

#### 18.4.3 值捕获 lambda → 闭包结构体

当 lambda 按值捕获外部变量时，编译器生成：
1. 一个**闭包结构体**（`struct Closure`），捕获变量的副本作为其字段；
2. 一个**独立的调用函数**，接受 `Closure*` 作为首个参数（环境指针）。

```rust
var base = 10;
var f = fn [base] (x: i32) -> i32 { return x + base; };
var result = f(5);
```

```c
// 闭包结构体
struct Closure__1 { int32_t base; };

// 调用函数（首参数为环境指针）
int32_t lambda__1(struct Closure__1* env, int32_t x) {
    return x + env->base;
}

// Vox 脱糖
int32_t base = 10;
struct Closure__1 __env_1 = { .base = base };
int32_t result = lambda__1(&__env_1, 5);
```

#### 18.4.4 引用捕获 lambda

当 lambda 通过引用捕获外部变量时，闭包结构体中存储指向外部变量的引用（`&T` 或 `&mut T`）而非值副本。

```rust
var counter = 0;
var f = fn [&mut counter] () -> i32 {
    counter = counter + 1;
    return counter;
};
```

```c
struct Closure__1 { int32_t* counter; };  // &mut T → T*

int32_t lambda__1(struct Closure__1* env) {
    *(env->counter) = *(env->counter) + 1;
    return *(env->counter);
}
```

引用捕获的 lambda 必须在其捕获变量的生命周期内使用，编译器在借用检查阶段验证这一点。

## 19. 附录

### 19.1 运算符优先级
（从高到低）

| 优先级 | 运算符 |
|--------|--------|
| 1（最高） | `()` 调用，`[]` 索引，`.` 字段访问 |
| 2 | `-`（一元负），`~`（按位取反），`*`（解引用） |
| 3 | `*` `/` `%` |
| 4 | `+` `-` |
| 5 | `<<` `>>` |
| 6 | `&` `\|` `^` |
| 7 | `<` `<=` `>` `>=` |
| 8 | `==` `!=` |
| 9 | `not`（关键字） |
| 10 | `and`（关键字） |
| 11（最低） | `or`（关键字） |

赋值运算符 `=` 优先级低于 `or`，右结合。  
范围运算符 `..` 优先级低于赋值运算符。  
一元取引用运算符 `&` / `&mut` 优先级同解引用（优先级 2）。

### 19.2 设计决策记录
- **为何没有 lambda**：lambda 需要闭包捕获机制，会引入隐式堆分配和复杂的生命周期管理，与"显式、无魔法"原则冲突。参数过多时，推荐使用结构体封装。（注：引用引入后，lambda 的设计已有坚实基础，见第 18.4 节。）
- **为何没有函数重载**：函数重载会增加符号查找的复杂性，且容易导致隐式行为。Vox 鼓励使用不同函数名或结构体参数。
- **为何文件即模块**：参考 Zig 的设计哲学。文件路径自动成为命名空间，无需手动管理 `namespace` 块，一个实体只出现一次。
- **为何需要显式 `return`**：保证求值语境的明确性。每条执行路径的意图都清晰可见。
- **为何引入 `global` 关键字**：强制程序员在函数内显式声明对模块级变量的访问，避免不小心遮蔽或无意修改全局状态。该关键字是纯编译期指令，零运行时开销。
- **为何禁止同作用域遮蔽**：防止旧变量隐式"消失"，保持作用域内名字绑定的单一性和可预测性。
- **为何选择转译至 C 作为第一步**：C 的语义与 Vox 高度匹配，且可直接利用成熟的 C 编译器获得原生性能与平台覆盖，是最快的原型验证路径。
- **为何不默认初始化变量**：默认初始化为 0 是一个隐式行为，会带来隐性运行时开销。Vox 强制程序员显式初始化所有变量，或在使用前显式赋值。
- **为何 I/O 极简**：`print` 不自动换行、不提供格式化占位符，所有行为由程序员显式控制。这符合"无魔法"原则。
- **为何使用 `&T` / `&mut T` 而非其他引用语法**：Rust 风格的引用语法是最清晰的选择——`&` 符号在视觉上即表示"借取地址"，与解引用 `*` 形成对称。`&T` 与 `&mut T` 的区别在类型层面显式表达，一眼可读。
- **为何不引入所有权转移（move 语义）**：`var b = a` 后 `a` 仍然有效——赋值不失效、传参不转移。这是"显式优于隐式"原则的体现：如果 `var b = a` 这行字面上没有说 `a` 会失效，那么 `a` 就不应该失效。需要转移语义时，使用显式的借用表达式 `&a` / `&mut a`。
- **为何引用必须遵守借用规则（原值冻结）**：`&mut T` 是对值的独占写入权——借出期间原值被冻结是引用的核心语义。"引用就真的是引用"，不是换了个名字的别名。参见第 10 节。
- **为何引入引用**：引用是现代语言编程的基石。对 Vox 而言，引用是必须而非选择——它为生命周期管理提供了安全可控的基础，也使 lambda 等高级特性的加入成为可能。与裸指针（`*T` / `*mut T`）并列，让程序员在安全和自由之间自由选择。
- **为何引入 lambda**：lambda 已成为现代语言的一等公民，在许多场景中具有不可替代的表达力。引用机制的引入使得 Vox 可以设计引用捕获的 lambda——捕获关系在类型层面清晰可见，生命周期由借用检查器保障。隐式之所以有害，根源在于不可控、不可预测；当引用的加入使生命周期问题变得可管理时，lambda 的加入便有了坚实的基础。
- **为何逻辑运算符选用 `and`/`or`/`not` 而非 `&&`/`||`/`!`**：消除"单 & 是位与还是逻辑与"的语义混淆。Python 方案将逻辑运算和位运算在语法层面彻底分离——逻辑运算用关键字 `and` `or` `not`，位运算用符号 `&` `|` `^` `~`——程序员无需依赖优先级记忆，代码扫读时语义一目了然。此设计借鉴自 Python，也是"显式优于隐式"原则的自然延伸。v0.5.0 起生效。
